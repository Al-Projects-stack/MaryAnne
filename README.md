# MaryAnne

System design and UML documentation for a booking and rewards web application.

## What I Learned

- Producing a complete UML and DFD artifact set for a booking and rewards system: use case, activity, sequence, context diagrams, and an ERD
- How to trace a single user flow (login, book appointment, earn rewards) through multiple diagram types and keep them consistent with each other
- The difference between a DFD Level 0 context diagram and a Level 1 functional decomposition, and what each is useful for
- Designing misuse case diagrams to surface security threats alongside normal system functionality
